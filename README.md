# 📅 프로젝트 관리 캘린더 (Project Manager)

본 프로젝트는 개인 및 팀의 프로젝트 일정과 할 일(To-Do)을 효율적으로 관리하기 위한 웹 애플리케이션입니다. 모든 데이터는 사용자의 브라우저 내부에 저장되어 별도의 서버 비용 없이 안전하게 사용할 수 있습니다.

## 🚀 주요 기능

### 1. 프로젝트 관리
- **상세 일정 관리**: 프로젝트 번호, 제목, 웹업 기간, 실사 기간, 최종 종료일 등을 관리.
- **자동 마감 설정**: 실사 종료일 입력 시 최종 종료일이 자동으로 맵핑됩니다 (수정 가능).
- **상태 추적**: 웹업 완료 여부 및 실사 시작 여부를 아이콘으로 한눈에 확인.
- **종료 알림**: 종료일이 지난 프로젝트는 상단 배너를 통해 알려주며, 종료 처리 후 3일 뒤에 자동으로 데이터가 삭제됩니다.

### 2. 할 일(To-Do) 관리
- **빠른 생성**: 날짜 선택 후 `Space` 키를 눌러 즉시 생성 가능.
- **중요도 설정**: 별점(1~5점) 시스템을 통한 우선순위 관리.
- **자동 이월**: 오늘 끝내지 못한 할 일은 다음 날로 자동 이월되어 잊지 않게 도와줍니다.

### 3. 스마트 캘린더 & 사이드 패널
- **다중 일정 표시**: 한 칸에 여러 개의 프로젝트와 할 일을 모두 표시.
- **일별 요약**: 날짜별 프로젝트(P)와 할 일(T) 개수를 요약 뱃지로 제공.
- **필터 기능**: 상세 세션에서 '프로젝트만', '할 일만' 혹은 '모두 보기' 필터링 가능.

### 4. 데이터 보안 및 백업
- **로컬 저장**: 서버로 데이터를 전송하지 않고 브라우저(`localStorage`)에만 저장하여 개인정보를 보호합니다.
- **백업/복원**: `.json` 파일 형태의 데이터 내보내기 및 불러오기 기능을 통해 기기 간 데이터 이동이 자유롭습니다.

## 🛠 기술 스택
- **Frontend**: Next.js (App Router), TypeScript, Tailwind CSS
- **Icons**: Lucide-React
- **Date Handling**: date-fns
- **Storage**: Browser LocalStorage

## 💻 실행 방법

### 로컬 개발 환경
1. 의존성 설치: `npm install` (client 폴더 내)
2. 실행: `npm run dev`
3. 접속: `http://localhost:3000`

### 정적 빌드 및 실행
1. 빌드: `build-static.bat` 실행
2. 로컬 서버 실행: `view-static-site.bat` 실행
